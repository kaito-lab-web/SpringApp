# Java ラムダ式 習得ガイド (Cheat Sheet)

## 1. ラムダ式とは？
**「メソッド（処理の内容）を、変数と同じようにシンプルに記述する書き方」**です。
従来の「匿名クラス（無名クラス）」という冗長な書き方を置き換え、コードを劇的に短く、読みやすくします。

---

## 2. 構文ルールまとめ

基本形: `(引数) -> { 処理 }`

### 【引数のルール】
* **引数が1つのとき**
  - ルール：カッコ `()` を省略可能
  - 例：`n -> n * 2`
* **引数が0個 または 2個以上のとき**
  - ルール：カッコ `()` が **必須**
  - 例：`(a, b) -> a + b`

### 【処理（中身）のルール】
* **処理が1行だけのとき**
  - ルール：`{}` と `return` を省略可能
  - 例：`(a, b) -> a + b`
* **処理が複数行にわたるとき**
  - ルール：`{}` と `return` が **必須**
  - 例：`(x, y) -> { int sum = x + y; return sum; }`

---

## 3. 重要用語解説

* **関数型インターフェース**
    * 抽象メソッドが**1つしかない**インターフェース（例：`Runnable`, `Consumer`）。
    * ラムダ式は、このインターフェースの「中身」として代入されます。
* **型推論 (Type Inference)**
    * コンパイラが文脈から型を自動判定する仕組み。
    * これにより `(int a) -> ...` ではなく `(a) -> ...` と短く書けます。
* **実質的 final**
    * ラムダ式の外側にある変数（ローカル変数）を使う場合、その変数の値が**後から変更されてはいけない**という制約です。

---

## 4. 実践：Stream API との組み合わせ
ラムダ式は、リスト操作（Stream API）で真価を発揮する

```java
List<String> lists = Arrays.asList("apple", "banana", "cherry");

// ① 全要素を表示 (forEach)
list.forEach(list -> System.out.println(list));

// ② 条件で絞り込んで加工 (filter & map)
List<String> results = list.stream()
    .filter(result -> result.length() > 5)   // 5文字より長いものに絞る
    .map(result -> result.toUpperCase())      // 大文字に変換する
    .toList();                      // 新しいリストにまとめる
